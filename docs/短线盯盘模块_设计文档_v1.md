# 短线盯盘模块 设计文档 v1 (草案，待评审)

> 目标用户：本系统使用者本人，短线打法 = **趋势回踩 (3-10 日)** + **打板/超短 (1-3 日)** 并行。
> 本文只做**设计**，不含实现。落地前请逐节评审，确认后再排实现顺序。
> 术语遵循 docs/TERMS.md（信号/预警/推送/报告/快照）。

---

## 0. 一句话定位

在现有"信号引擎 + 多扫描器"体系上，**补一个市场级"情绪温度"聚合层 + 一个短线盯盘仪表盘页面**。
现有体系强在「个股买卖点 + 持仓风控」，弱在「今天敢不敢干、干哪个方向、情绪在周期哪一段」——这正是短线的命门。

---

## 1. 短线数据全景（决策链分层 + 现有覆盖盘点）

按盯盘时的真实决策顺序分 4 层。✅=已有可直接复用 / ⚠️=底层能取需聚合 / ❌=需新建。

### 第 0 层 · 大盘情绪温度（决定今天能不能干、仓位多重）
| 数据 | 短线用途 | 现状 | 来源 |
|---|---|---|---|
| 指数实时涨跌/急跌速度 | 跳水保命 | ✅ | `plunge_detector` |
| 涨跌家数 / 涨停跌停数 | 赚钱效应温度计 | ✅ | `PLUNGE_BREADTH`、market_overview 快照 |
| **封板率 / 炸板率** | 情绪退潮最灵敏指标 | ⚠️ 需市场级聚合 | 东财涨停池接口 |
| **连板高度梯队**（最高几板、各高度家数） | 情绪周期定位（冰点/启动/高潮/退潮） | ❌ | 东财涨停池 / 概念标签 |
| **昨涨停今日溢价** | 接力资金赚不赚钱→明天敢不敢打 | ❌ | sectors.py 已暴露 `昨日涨停` 标签 + quotes 现价 |

### 第 1 层 · 板块与题材（决定干哪个方向）
| 数据 | 用途 | 现状 | 来源 |
|---|---|---|---|
| 板块资金净流入排行 | 找主线/进攻方向 | ✅ | `capital_inflow_scanner`、`sector_leader`、`attack_direction_analyst` |
| 题材热度/人气 | 主线持续性 | ✅ | `popularity_refresher`、`trading_concepts` |
| 板块梯队（龙头/跟风） | 选最强不补弱 | ✅ 部分 | `sector_leader` |

### 第 2 层 · 个股盘口与异动（决定买哪只、什么价）
| 数据 | 用途 | 现状 | 来源 |
|---|---|---|---|
| 分时/均价/量比 | 即时强弱 | ✅ | `intraday`、`quote_refresher`、`stock_extra` |
| **集合竞价**（金额/高低开/竞价异动） | 超短第一战场 | ✅ | `auction_summary_analyst` |
| 个股大单/主力净流入 | 资金真进场否 | ✅ | `fetcher/big_money.py` |
| 换手率/流通市值/量比 | 活跃度承接 | ✅ | `stock_extra`（turnover/volume_ratio/free_cap） |
| **实时涨速榜 / 异动榜**（全市场） | 抓盘中启动 | ❌ 现有 speed 仅自选池 | quotes f11 全市场拉取 |

### 第 3 层 · 买卖点信号与持仓风控（决定动手/止损）
| 数据 | 现状 |
|---|---|
| 买点：弱势极限 / 主升浪回踩MA10启动(S3_BUY) / STRONG_START | ✅ 完整 |
| 卖点：破 MA5/10/20 / 止盈 / 追踪止盈 / 盈亏比 / 时间止损 | ✅ 完整 |
| 持仓浮亏分级 5/8/10% | ✅ PLOSS 系列 |
| 信号胜率复盘 | ✅ `signal_outcome_backfill` |

**结论**：第 2、3 层基本完备。本次设计聚焦 **第 0 层（市场情绪聚合）** 为主，**第 2 层异动榜** 为辅。

## 2. 两种打法的数据侧重差异（同一套数据，两套视角）

| 维度 | 趋势回踩 (3-10日) | 打板/超短 (1-3日) |
|---|---|---|
| 核心问题 | 主线票回踩到位没有 | 今天情绪能不能接力 |
| 第 0 层权重 | 中（过滤极端风险） | **极高**（情绪周期决定生死） |
| 关键指标 | 均线结构、量能、强势质量评分 | **封板率/炸板率、连板高度、昨涨停溢价** |
| 买点信号 | S3_BUY 回踩MA10、弱势极限 | 竞价异动、首板/连板梯队、实时涨速 |
| 持有周期 | 看 MA10/MA20 破位卖 | 看分时、不破板/破板即走 |

→ 设计上：**底层数据一份，前端用「打法切换」呈现两套盯盘视图**（详见 §5）。

## 3. 新增数据模型（cfzy_ 前缀，对齐现有 DB 规约）

### 3.1 `cfzy_sys_emotion_snapshot` — 市场情绪温度快照
定点（盘中每 N 分钟 + 收盘）落库一份，**只供前端读，不参与触发**（符合"快照"定义）。

| 字段 | 含义 |
|---|---|
| `id` / `captured_at` / `trade_date` | 主键 / 采集时刻 / 交易日 |
| `limit_up_count` / `limit_down_count` | 涨停 / 跌停数 |
| `seal_rate` | 封板率 = 涨停封住数 /（涨停封住 + 炸板） |
| `broken_board_count` | 炸板数 |
| `up_count` / `down_count` / `flat_count` | 涨/跌/平家数 |
| `highest_board` | 当日最高连板高度 |
| `board_ladder_json` | 连板梯队 `[{height:5,count:1},{height:4,count:2},...]` |
| `yest_limit_up_premium` | 昨涨停今日平均涨幅（接力溢价，情绪核心指标） |
| `emotion_phase` | 派生：冰点/修复/启动/高潮/退潮（规则见 §4.2） |
| `up_amount_ratio` | 涨停板块成交占比（可选） |

### 3.2 盘中异动榜 — **仅自选池范围**（已确认收窄）
范围收窄为**自选池 + 持仓票**（不全市场拉取，规避风控与数据量）→ 复用现有 `quote_refresher` 对池子的轮询结果，按 speed(f11) 排序即可，**无需新表、无需新外部请求**，纯内存/接口聚合。

> 复盘需求：`emotion_snapshot` 按 trade_date 留存，可做"情绪周期 vs 自己交易胜率"的回测——和现有 `signal_outcome_backfill` 打通。

## 4. 后端设计（FastAPI 分层，对齐现有 service/router/repo 三层）

### 4.1 数据采集层 — 复用现有 fetcher，补 1 个市场级接口（多源 fallback）
- **新增** `backend/fetcher/limit_pool.py`：拉涨停池，**对齐 klines.py 的"三源 fallback + 降级"模式**（东财对 prod IP 有风控史）：
  1. **主源** 东财 `push2ex.eastmoney.com/.../getTopicZTPool`（含连板数、封板时间、炸板标记）→ 一次请求即可算 封板率/炸板率/连板梯队/最高板。
  2. **备源** 同花顺涨停池（`d.10jqka.com.cn` / `data.10jqka` 涨停接口）→ 字段对齐主源。
  3. **降级** 两源皆失败时，从现有全市场 `quotes` 估算：按涨跌幅阈值（±9.8%/±19.8%/±29.8% 按板块）数涨停/跌停 → **涨停数/跌停数仍可得，但封板率/连板高度取不到 → 这些字段写 null**，`emotion_refresher` 据此把 `emotion_phase` 标记为「数据降级·不可判」。
  4. **最终降级** 全失败 → 读上一档 `cfzy_sys_emotion_snapshot`，前端标「stale + 降级」。
  - 数据源状态打到现有 `api_metrics`（与 ApiHealthIndicator 一致），前端 Dashboard 显示当前用的是哪一档。
- **复用** `sectors.get_stock_concepts` 已暴露的 `昨日涨停/昨日连板/昨日炸板` 标签
  → 取昨涨停代码集合，配合 `quotes` 现价算「昨涨停今日溢价」。
- **复用** `quotes`(speed=f11) 做异动榜；`big_money` 做个股资金。

### 4.2 计算/聚合层 — 新增 service
- **新增** `backend/services/emotion_refresher.py`：
  - 被 scheduler 定时调用（**盘中每 3 分钟** + 09:30/11:30/15:00 定点），算出 §3.1 全字段，写 `cfzy_sys_emotion_snapshot`。
  - `emotion_phase` 派生规则（初版，可调）：
    - **冰点**：最高连板高度 ≤ 3 且 昨涨停溢价<0（用户盘感校准 v1）
    - **修复**：昨涨停溢价由负转正、炸板率下降
    - **启动**：最高板抬升、封板率>80%
    - **高潮**：连板梯队完整(高度≥5)、涨停数高位
    - **退潮**：封板率骤降、炸板率升、昨涨停溢价转负
  - 阈值集中到 `signal_engine_config` 风格的配置，可前端调（对齐现有 SignalConfig 习惯）。
- **信号化退潮（已确认要做，P2）**：把"退潮"做成一条 `regime` 组信号 **`PLUNGE_EMOTION_EBB`**（命名对齐 v1.7.181 前缀规约：regime 组 / direction='plunge' 走 `PLUNGE_` 家族）。
  - 强档（priority=3，盘中实时**推送企微**），在 `signal_specs.py` 三处登记（group=regime / priority=3 / timing=intraday）。
  - 退潮判据复用 §4.2 的 `emotion_phase` 结果：phase 由非退潮→退潮的**那一刻**触发一次（去抖，避免同日反复推）→ 复用现有 `alert_throttle`。
  - 中文名（对齐规约）：**「大盘预警·情绪退潮」**。

### 4.3 接口层 — 新增 router
- **新增** `backend/routers/emotion.py`：
  - `GET /api/emotion/current` → 最新一档情绪快照
  - `GET /api/emotion/history?date=` → 当日时间序列（画情绪曲线）
  - `GET /api/emotion/movers` → 异动榜（第二期）
- 鉴权/限频沿用现有 router 中间件。

## 5. 前端设计（Vue3 + TS，对齐现有 views/components 结构）

### 5.1 新增页面 `EmotionDashboardView.vue`（短线情绪盯盘）
顶部「打法切换」Tab：**[趋势回踩] / [打板超短]**，切换只改卡片排布与默认展开项，数据同源。

布局（卡片化，浅色主题，对齐现有 UI 规范）：
1. **情绪温度计**（最显眼）：emotion_phase 大字 + 封板率/炸板率/连板梯队条形图 + 昨涨停溢价。
2. **连板梯队**：高度×家数的金字塔，点击下钻到该高度个股。
3. **方向**：板块资金流 top（复用现有 sector 组件/接口）。
4. **异动/涨速榜**（仅自选池+持仓，第三期）：池子内按涨速实时滚动。
5. **我的池子叠加**：现有自选池/信号叠加在情绪背景上（复用 PoolView 组件）。

- **打板视图**默认突出 1、2、4；**回踩视图**默认突出 3 + 强势质量评分 + S3_BUY 信号。

### 5.2 复用
- 图表走现有 `components/chart`；卡片/加载/空态走 `components/common`。
- 情绪曲线（当日 emotion 时间序列）用现有 sparkline/chart 能力。

## 7. 已确认决策（2026-05-30 评审）

| # | 问题 | 决策 | 已并入 |
|---|---|---|---|
| 1 | 涨停池数据源 | **需备源/降级** → 东财主 + 同花顺备 + 全市场行情估算降级 + 上档快照兜底 | §4.1 |
| 2 | 情绪阶段阈值 | 冰点判据改为 **最高连板 ≤ 3** | §4.2 |
| 3 | 采集频率 | **盘中每 3 分钟足够** | §4.2 |
| 4 | 退潮是否信号化 | **要做，强档推企微**，signal_id=`PLUNGE_EMOTION_EBB` | §4.2、P2 |
| 5 | 异动榜范围 | **仅自选池 + 持仓**，不全市场拉取 | §3.2、§5.1、P3 |

### 数据源验证（已完成 2026-05-30，脚本 `backend/scripts/probe_limit_pool.py`，取 5/29 数据）
**两源均可用、均无需 cookie，风控风险低。** 字段映射：

| 设计指标 | 东财主源 (push2ex getTopicZTPool / ZBPool) | 同花顺备源 (10jqka limit_up_pool) |
|---|---|---|
| 涨停数 | 涨停池 len=49 | info len=49 |
| 炸板数 | 炸板池 getTopicZBPool len=38 | `open_num` 开板次数 |
| 封板率 | 49/(49+38)=56% | `limit_up_suc_rate` 直接给 |
| 连板高度 | `lbc` 连板数 / `zttj.days` | `high_days`("首板"/天数) 语义更清晰 |
| 封单资金 | `fund` | `order_amount` / `order_volume` |
| 行业/题材 | `hybk` 行业 | `reason_type` **涨停原因(题材)** 更可喂题材识别 |
| 板型 | — | `limit_up_type`(换手板/一字板) |
| 跌停数 | getTopicDTPool len=0(当日无) | — |

- **价格单位注意**：东财 `p`/`ltsz` 等是 ×100 整数(如 19570=195.70)，实现时按 fetcher 既有惯例 `_safe_num` 处理。
- **时间字段**：东财 `fbt`=92500(09:25:00 整数)，同花顺是 unix 秒级时间戳，两源需归一。
- §4.2 情绪阶段其余阈值（封板率/炸板率边界）按实盘观察迭代，配置可前端调。
