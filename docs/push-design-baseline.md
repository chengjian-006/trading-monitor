## 一、五大家族（header 颜色 = 这条是干嘛的）

A 股语境定色：红 = 涨 = 机会，绿 = 跌 = 离场。每张卡归且只归一个家族。

| 家族 | header | 标题 emoji 词表 | 覆盖 |
|---|---|---|---|
| 机会 | red | 📈 买入 / 🔥 二波 / ⚡ 机会 | 买点 8 张 + 预增榜 |
| 离场 | green | 📉 卖出 / 🛡️ 减仓 | 卖出/止损/尾盘破位 5 张 |
| 风险 | orange(谨慎) / red(空仓·急跌) | 🚨 / ⚠️ / 📛 / 🦢 | 大盘风控/板块共振/黑天鹅 8 张 |

唯一允许越级：**纪律升级卡**（止损连续未执行）从绿越级到红——它已从「建议」升格为「警报」。

## 二、五区骨架（任何卡从上到下只有这五区，缺哪区空哪区，顺序不变）

0. **大盘风险横幅**（全局自动注入，带 since 锚点，v1.7.629/630 已上线，单卡不用管）
1. **结论区（必有）**：一句话讲完——标的 + 发生了什么 + 最关键的一个数。信号/情报类用 KPI 三栏承担此区
2. **数据区（可选）**：图形词汇表元素（见下）或 md_table ≤3 列
3. **行动建议（必有）**：`👉 **≤20字大白话**`，动词开头；情报类给一句话定性
4. **折叠详情（可选）**：collapsible_panel 默认收起——触发明细、方法论、长名单、术语解释全下沉
5. **快捷动作行（可选）**：固定词表 `📈 看分时图 · 🔕 静音 · ✅ 已卖出`，永远最后一行

## 三、图形词汇表（真机验证定稿）

### 大杀器：原生图表（chart 组件，群机器人 webhook 实测可渲染）
```python
{'tag': 'chart', 'aspect_ratio': '2:1', 'chart_spec': {
    'type': 'line',  # 或 'bar'
    'data': [{'id': 'd', 'values': [{'x': '07-16', 'y': 40}, ...]}],
    'xField': 'x', 'yField': 'y',
    'point': {'visible': True},           # 折线带点
    'color': ['#D83931'],                 # 红涨绿跌
    'axes': [{'orient': 'left', 'zero': False}],   # y 轴不从 0 起(波动更明显)
    'padding': {'top': 2, 'bottom': 0, 'left': 4, 'right': 8},
}}
```
硬规则：**必须 `aspect_ratio: '2:1'` 压扁**（默认高度占大半屏）；折线 y 轴 `zero: False`；每卡图表 ≤2 个。

### 标配 8 件
| 元素 | 写法 | 硬规则 |
|---|---|---|
| KPI 三栏大字 | column_set 3 列，md 元素 `text_size: 'heading'` + 灰色小标签行 | 恰好 3 栏；数字层 heading、标签层普通灰 |
| 强度条 | `<font color='red'>▰▰▰▰▰</font><font color='grey'>▱▱▱</font> **62%**` | **≤8 格**；一行只放一组数据 |
| ✅ 触发清单 | `✅ 条件 **实测值**（要求 门槛）` 竖排逐行 | 触发链标准形态（二波卡 v1.7.625 同款） |
| 战绩热图 | `🟩🟩🟥🟩⬜ 7胜2负`（🟩赢 🟥亏 ⬜无） | ≤10 格 |
| 温度计 | `<font color='blue'>▰▰</font><font color='orange'>▰▰▰</font><font color='red'>▰▰</font><font color='grey'>▱▱▱</font> **68°** 偏热` | 分段变色，总 ≤10 格 |
| 多空条 | `<font color='green'>跌33 ◀▰▰▰</font>｜<font color='red'>▰▰▰▶ 涨40</font>` | **每边 ≤3 格**保一行 |
| 灯串 | `🟢🟢🟢🟡 3正常 1要看` | **≤8 盏**，多了改计数式 `🟢9 🟡2` |
| heading 大字行 | md 元素 `text_size: 'heading'` | 单行核心结论/合计数用 |

### 弃用（真机验证淘汰，禁止使用）
- **横排步进灯** `🟢 A → 🟢 B → 🟢 C`：手机必折行、箭头断开 → 用 ✅ 竖排清单
- **字符走势** `▁▂▃▅▇`：手机渲染成黑粗块 → 用原生折线图
- **字符柱状图** `▉▉▉`：被原生柱状图取代

### 家族图形配置
- **信号卡** = KPI 三栏(现价/涨幅/排名) + 胜率条 + 战绩热图 + ✅ 触发清单
- **情报卡** = KPI 三栏 + 趋势折线 + 梯队柱状 + 多空条
- **风险卡** = 温度计 + 多空条 + since 横幅(自动)
- **系统卡** = 灯串 + 异常项列表

## 四、六条细节规范

1. **标题公式**：`emoji 事件名 · 标的(代码)`；多标的 `emoji 事件名 · N只`；时间由标题栏自动拼，**正文永不写时间**（v1.7.635）；测试卡后缀`（测试样例）`
2. **数字**：实测值加粗、门槛常规体（`**1.9×**（要求 ≥1.8×）`）；涨跌带色带号 红`+2.7%`/绿`-1.8%`；金额中文单位 `3.2 亿`；价格 `¥58.39`；禁 ①②③（宽度歧义）
3. **语言**：大白话禁代号（弱势极限 ≠ S2）；建议 ≤20 字动词开头；解释"为什么"用 ✅ 清单式
4. **表格**：md_table（禁原生 table_element，手机截断）；≤3 列（手机基准 2 列）；名单 >8 行放 Top5+「等N只」，全量进折叠
5. **降噪**：正文（0~3 区）≤10 行；一卡一事；同事件多标的合并；系统运维类降级日终汇总
6. **落地纪律**：新推送必须走 card_kit 构造器；预览页（lark_templates.py）与真实推送 1:1 同步；本基线改版需真机验证后升版本号
