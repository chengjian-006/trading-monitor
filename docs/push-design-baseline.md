# 观潮推送设计基线 v1.1（2026-07-16 第三轮真机验证定稿）

飞书推送卡的统一设计标准。**后续所有推送改动一律以本基线为准**；新推送必须走 `card_kit.py` 构造器（已建, backend/services/card_kit.py, v1.7.639），不再手拼卡片 JSON。可视化样稿（仿飞书渲染的完整设计稿）见 Claude Artifact `596903a2`。

所有图形元素均经手机端飞书**三轮真机验证**（前两轮 2026-07-16 上午定 v1.0；第三轮同日晚，对标 4 路调研后补验 chart media 覆盖/摘要/副标题标签/深色模式/聚合卡表格版式，验证卡在推送群里留档）。

## 一、五大家族（header 颜色 = 这条是干嘛的）

A 股语境定色：红 = 涨 = 机会，绿 = 跌 = 离场。每张卡归且只归一个家族。

| 家族 | header | 标题 emoji 词表 | 覆盖 |
|---|---|---|---|
| 机会 | red | 📈 买入 / 🔥 二波 / ⚡ 机会 | 买点 8 张 + 预增榜 |
| 离场 | green | 📉 卖出 / 🛡️ 减仓 | 卖出/止损/尾盘破位 5 张 |
| 风险 | orange(谨慎) / red(空仓·急跌) | 🚨 / ⚠️ / 📛 / 🦢 | 大盘风控/板块共振/黑天鹅 8 张 |
| 情报 | blue | 📊 | 竞价/复盘/日报/轮动/晚报 10 张 |
| 系统 | grey | ⚙️ | EOD复核/数据源/任务失败 6 张 |

唯一允许越级：**纪律升级卡**（止损连续未执行）从绿越级到红——它已从「建议」升格为「警报」。

**解除卡（v1.1 新增）**：状态型预警（情绪冰点/退潮/空仓预警等）结束时发灰 header 解除卡收尾（形态见第五节），灰 = 中性收尾不抢注意力。

## 二、五区骨架（任何卡从上到下只有这五区，缺哪区空哪区，顺序不变）

0. **大盘风险横幅**（全局自动注入，带 since 锚点，v1.7.629/630 已上线，单卡不用管）
1. **结论区（必有）**：一句话讲完——标的 + 发生了什么 + 最关键的一个数。信号/情报类用 KPI 三栏承担此区
2. **数据区（可选）**：图形词汇表元素（见下）或全短列 md_table
3. **行动建议（必有）**：`👉 **≤20字大白话**`，动词开头；情报类给一句话定性
4. **折叠详情（可选）**：collapsible_panel 默认收起——触发明细、方法论、长名单、术语解释、**表格挤不下的长值（价格等）**全下沉
5. **快捷动作行（可选）**：固定词表 `📈 看分时图 · 🔕 静音 · ✅ 已卖出`，永远最后一行；**≤3 个动作**；危险语义色仅用于「已卖出」类不可逆操作

**区外标配（v1.1 新增）——自定义摘要**：每卡 `config` 必带摘要，内容 = 标题公式同款「标的 + 事件 + 关键数字」（如 `拓普集团 601689 缩量突破 ¥58.39 +2.7%`）。真机验证：锁屏横幅/会话列表直接显示摘要正文而非"[卡片]"占位，很多决策锁屏一眼完成。

```python
'config': {'wide_screen_mode': True, 'width_mode': 'fill',
           'summary': {'content': '拓普集团 601689 缩量突破 ¥58.39 +2.7%'}}
```

## 三、图形词汇表（真机验证定稿）

### 大杀器：原生图表（chart 组件，群机器人 webhook 实测可渲染）
```python
{'tag': 'chart', 'aspect_ratio': '2:1', 'chart_spec': {
    'type': 'line',  # 或 'bar'
    'media': [],                          # 必须！清空平台默认响应式(移动端会被拉回1:1大方图)
    'data': [{'id': 'd', 'values': [{'x': '07-16', 'y': 40}, ...]}],
    'xField': 'x', 'yField': 'y',
    'point': {'visible': True},           # 折线带点
    'color': ['#D83931'],                 # 红涨绿跌
    'axes': [{'orient': 'left', 'zero': False}],   # y 轴不从 0 起(波动更明显)
    'padding': {'top': 2, 'bottom': 0, 'left': 4, 'right': 8},
}}
```
硬规则：**必须 `aspect_ratio: '2:1'` 压扁 + `chart_spec` 内 `'media': []`**（第三轮真机 A/B 对照：不带 media 的卡在手机端被平台默认 media query 拉回 1:1 大方图；media 只能放 `chart_spec` 内，放 chart 元素层 API 直接拒收 200621）；折线 y 轴 `zero: False`；每卡图表 ≤2 个。

### 标配 8 件
| 元素 | 写法 | 硬规则 |
|---|---|---|
| KPI 三栏大字 | column_set 3 列，md 元素 `text_size: 'heading'` + 灰色小标签行 | 恰好 3 栏；数字层 heading、标签层普通灰 |
| 强度条 | `<font color='red'>▰▰▰▰▰</font><font color='grey'>▱▱▱</font> **62%**` | **≤8 格**；一行只放一组数据 |
| ✅ 触发清单 | `✅ 条件 **实测值**（要求 门槛）` 竖排逐行 | 触发链标准形态（二波卡 v1.7.625 同款） |
| 战绩热图 | `🟩🟩🟥🟩⬜ 7胜2负`（🟩赢 🟥亏 ⬜无） | ≤10 格 |
| 温度计 | `<font color='blue'>▰▰</font><font color='orange'>▰▰▰</font><font color='red'>▰▰</font><font color='grey'>▱▱▱</font> **68°** 偏热` | 分段变色，总 ≤10 格 |
| 多空条 | `<font color='green'>跌33 ◀▰▰▰</font>｜<font color='red'>▰▰▰▶ 涨40</font>` | **每边 ≤3 格**保一行 |
| 灯串 | `🟢🟢🟢🟡 3正常 1要看` | **≤8 盏**，多了改计数式 `🟢9 🟡2` |
| heading 大字行 | md 元素 `text_size: 'heading'` | 单行核心结论/合计数用 |

**色阶渐变（v1.1 新增可选写法）**：`<font>` 支持官方色阶枚举（如 `red-300/500/700`），真机深浅两端渐变分层清晰，做单色渐变强度条/热图比拼主色更细腻，深色适配由平台兜底。全套字符图形经深色模式真机复核各档可辨。

### 弃用（真机验证淘汰，禁止使用）
- **横排步进灯** `🟢 A → 🟢 B → 🟢 C`：手机必折行、箭头断开 → 用 ✅ 竖排清单
- **字符走势** `▁▂▃▅▇`：手机渲染成黑粗块 → 用原生折线图
- **字符柱状图** `▉▉▉`：被原生柱状图取代
- **column_set 做多行对齐表**（v1.1 淘汰）：栏内换行后左右行错位
- **原生 table_element**（维持淘汰，v1.1 复验）：即便全短值+百分比列宽，彩色标签列在手机端仍截断

### 家族图形配置
- **信号卡** = KPI 三栏(现价/涨幅/排名) + 胜率条 + 战绩热图 + ✅ 触发清单
- **情报卡** = KPI 三栏 + 趋势折线 + 梯队柱状 + 多空条
- **风险卡** = 温度计 + 多空条 + since 横幅(自动)
- **系统卡** = 灯串 + 异常项列表

## 四、细节规范

1. **标题公式**：`emoji 事件名 · 标的(代码)`；多标的 `emoji 事件名 · N只`；时间由标题栏自动拼，**正文永不写时间**（v1.7.635）；测试卡后缀`（测试样例）`
2. **header 副标题与标签（v1.1 新增）**：主标题只留「emoji 事件 · 标的」瘦身防折行；模型名/强度档/归因词放 `text_tag_list` 彩色小标签（**≤3 个**，需飞书 V6.11+）；口径说明/时间线放 `subtitle`（单行超长省略，务必短）。真机验证渲染正常
3. **数字**：实测值加粗、门槛常规体（`**1.9×**（要求 ≥1.8×）`）；涨跌带色带号 红`+2.7%`/绿`-1.8%`；金额中文单位 `3.2 亿`；价格 `¥58.39`；禁 ①②③（宽度歧义）
4. **语言**：大白话禁代号（弱势极限 ≠ S2）；建议 ≤20 字动词开头；解释"为什么"用 ✅ 清单式
5. **表格（v1.1 收窄）**：只允许 **全短列 md_table**——每格都是短值（股票名/百分比/短标签级别），任何长值（价格、长文本）**下沉折叠区**；**列序 = 重要列前置，最后一列放最短、截了不心疼的值**（手机端 md_table 无列宽控制、超宽必截尾列）；名单 >8 行放 Top5+「等N只」，全量进折叠。带长值的多列场景整体改行式清单或下沉
6. **降噪**：正文（0~3 区）≤10 行；一卡一事；同事件多标的合并（聚合卡，见第五节）；系统运维类降级日终汇总
7. **落地纪律**：新推送必须走 card_kit 构造器；预览页（lark_templates.py）与真实推送 1:1 同步；本基线改版需真机验证后升版本号

## 五、标准卡型（v1.1 新增两型）

### 聚合卡（风暴合并，真机定稿 = 全短列表格形态）
同族信号短窗口内（90 秒）凑够 ≥3 条时合并为一张，普跌日防轰炸。结构：

- header：orange（风险聚合）或对应家族色；标题 `🚨 集中触发 · 自选N只 事件名`；标签放共因（如「板块共振」）；副标题放合并窗口 `10:42~10:44 合并推送`
- 归因一行：`**归因**：大盘急跌 -1.8%，光伏板块共振下杀（板块内 5/7 只自选同跌）`
- 全短列 md_table：`| 股票 | 跌幅 | 信号 |`（跌幅绿字加粗；现价等长值进折叠区「现价明细」）
- 👉 建议（如 `板块共振跌，禁抄底，等企稳再看`）
- 折叠：合并口径说明 + 长值明细

### 解除卡（状态型预警闭环）
状态型预警（情绪冰点/退潮/空仓预警）结束时收尾，与首发卡形成闭环。结构：

- header：grey；标题 `✅ 预警解除 · 预警名`；标签「已解除」；副标题带时间线 `07-08 09:30 发布 → 今日解除，生效 6 个交易日`
- 解除条件一行（写明是哪个条件解除的）：`✅ 解除条件：涨停家数5日均 **48** ≥ 45（连续 2 日，防贴线反复）`
- 生效期间小结一行（期间指数/关键指标变化）
- 👉 建议（如 `冰点解除，可逐步恢复试仓`）
- 防抖硬规则：解除判定需**超缓冲带或持续 N 个周期**，防贴线反复横跳；一次性事件卡（买卖点）不发解除

## 附录：飞书 webhook 硬约束速查

- 请求体 ≤20KB；频控 100 次/分、5 次/秒
- 单卡 ≤200 元素；chart ≤5 个/卡（官方），本基线收紧为 ≤2
- `aspect_ratio` 仅 4 档：1:1 / 2:1 / 4:3 / 16:9；不写 media 时平台自动加响应式（PC 16:9/移动 1:1）
- header `text_tag_list` ≤3 个标签（V6.11+）；subtitle 单行超长省略
- 群机器人 webhook 单向推送：按钮只能 URL 跳转（无回调）；img/header 图标需 img_key 上传通道，webhook 不可用
- 卡片 JSON 2.0 需客户端 7.20+；chart 的 VChart v1.12.3 需 7.27+
- md_table 无列宽参数；原生 table_element 有 `width` 百分比但彩色标签列手机端仍截断（已弃用）
