# AI 顾问（交易教练 + 个股研判卡）设计

- 日期：2026-07-19
- 状态：设计已确认，待写实现计划
- 关联：[[project_trade-analysis-feature]] [[project_model-winrate-feature]] [[project_risk-announcement-monitor]] [[project_website]]（工具口径红线）

## 1. 目标与边界

给 A 股交易者两个基于 AI 的功能，**共同原则：把用户自己系统里已有的客观数据嚼成大白话，摆事实、不预测、不喊买卖**（与官网红线一致）。

- **AI 交易教练（私人复盘）**：定期 + 按需，用用户真实交割单/回合/听模型对比数据，指出其本人的交易习惯与成绩规律。
- **个股 AI 研判卡**：按需，把某只票的信号历史、同形态胜率、财务红旗、板块强弱、持仓成本综合成一段"这票现在什么位置、风险点在哪"的大白话。

**明确不做（YAGNI + 红线）**：不预测涨跌方向、不喊买卖点、不报目标价、不承诺胜率；LLM 不自己算数字、不碰原始数据。

## 2. 核心架构（方案 A：厚后端 + 薄 LLM）

一条共用链路：`采集数据 → 拼"事实清单"(纯后端确定性算, 数字都真) → LLM 只把清单写成人话 → 渲染/推送`。

新增模块目录 `backend/services/ai_advisor/`（独立成模块，不塞进现有大文件）：

| 文件 | 职责 | 碰 LLM? | 碰网络? |
|---|---|---|---|
| `ai_client.py` | 统一 LLM 调用封装。provider 从 config 选，对外仅 `async narrate(system_prompt, fact_sheet) -> str` | ✅ 唯一 | ✅ |
| `coach_facts.py` | 交易教练事实清单构造器（纯函数） | ❌ | ❌（喂库数据） |
| `stock_facts.py` | 研判卡事实清单构造器（纯函数） | ❌ | ❌（喂库数据） |
| `trade_coach.py` | facts → narrate → 组装复盘结果，供路由/周任务调 | 间接 | — |
| `stock_review.py` | facts → narrate → 组装研判卡，供路由调 | 间接 | — |

**关键不变式**：LLM 只在 `ai_client.narrate` 一处出现；两个 facts 构造器纯确定性、可单测、不依赖网络。"数字真不真"完全由后端保证，与 LLM 无关。

### 2.1 ai_client（provider 可配）

- config 读 `ai_advisor_provider`：`deepseek`（复用现有 `ai_analyst` 的 OpenAI 兼容客户端）/ `claude`（anthropic 兼容端点，用 config 的 `anthropic_api_key` / `ai_base_url`）。
- 同步 SDK 调用走 `asyncio.to_thread`，不阻塞事件循环（同 `ai_analyst.generate_report` 现有做法）。
- 失败/超时/返回空 → 返回 `None`，由上层降级。

## 3. AI 交易教练

### 3.1 事实清单 `coach_facts.build(user_id, start, end)`

纯确定性，四类指标（数据源为 trade_rounds/交割单/持仓/model_winrate）：

1. **买卖习惯坏毛病**：追高买入占比（买入价距当日高点%）、卵巢买/追涨买分布、卖飞幅度（卖出后 N 日续涨%）、输家扛单天数、止损纪律（跌破止损位后实际砍在第几天）。每项给数字 + 与"好的那批"对照。
2. **听模型 vs 自作主张**：回合按"是否归因到买点模型"分两组，对比胜率/单笔均收益（复用 v1.7.209 口径）。
3. **模型归因成绩**：实盘走得最多/最赚/最亏的买点模型，各与全市场回测胜率（cfzy_biz_model_winrate）对比。
4. **盈亏/持仓周期规律**：持仓天数分布、单笔盈亏分布、赢家平均持有 vs 输家扛单时长。

输出：结构化 dict（每个指标含数值 + 对照 + 简短口径说明）。

### 3.2 交付

- **定期推送**：新增定时任务 `run_trade_coach_weekly`，**每周日晚**生成"近一月交易复盘"推一张飞书卡，走现有推送闸门（`send_wechat_signal` 统一入口 + 偏好开关）。失败走现有任务失败告警链路。
- **页面按需**：新增"交易复盘"页（挂"策略绩效"菜单组），选时间段点生成。展示事实清单（图表/表格）+ LLM 大白话复盘。
- **缓存**：生成结果落库，同用户 + 同区间当天不重复调 LLM。

## 4. 个股 AI 研判卡

### 4.1 事实清单 `stock_facts.build(code, user_id)`

纯确定性，四源：

1. **信号历史 + 同形态胜率**：该票历史买卖点、当前是否临近买点（复用临近买点榜口径）、所属买点模型的全市场同形态历史次日/3日真实涨跌分布（摆事实非预测）。
2. **财务红旗 + 公告风险**：复用黑天鹅监控（财务红旗9项 + 商誉/质押/减持 + 公告硬信号）。
3. **板块强弱 + 题材热度**：所属行业/题材当前强弱名次、涨停题材热度、（若持仓）在最热题材内位置。
4. **成本/持仓上下文**：**仅当该票在用户持仓**才带——成本、浮盈、建仓买点、距前高/支撑位；非持仓票留空。

### 4.2 交付

- **触发**：股票池/个股详情加"AI 研判"按钮 → 后端拼清单 → LLM 写研判 → 弹研判卡。
- **成本控制**：
  - 按 `股票+交易日` 缓存，当天已生成直接返回。
  - 每人每日研判上限 **200 次**（`ai_advisor_daily_cap`），防手滑狂点。
  - 事实清单可脱离 LLM 单独显示；LLM 段可单独失败降级。
- **展示**：一张卡——顶部事实速览（结构化）+ 中间 LLM 大白话研判 + 底部固定红线声明。

## 5. 红线护栏

- LLM 系统提示词硬约束：只准复述事实清单里的数字，**禁止自己算数、禁止预测涨跌、禁止给买卖建议**；输出数字须能在清单找到出处。
- 与官网同红线：不承诺胜率、不喊买卖、不报目标价。
- 研判卡/复盘卡底部固定声明："客观历史数据 + AI 归纳，非投资建议、不预测涨跌"。
- 方案 A 天然保证：数字全后端算，LLM 措辞飘也编不出假胜率。

## 6. 错误处理

- LLM 挂/超时/空 → 照常显示事实清单（硬数据），只少那段叙述，标"AI 叙述暂不可用"。功能不因 LLM 失败全瘫。
- provider 可配 + 失败降级；研判卡当日缓存 + 每人 200/天上限；周任务失败走现有告警链路。

## 7. 测试

- `coach_facts.build` / `stock_facts.build` 纯函数：喂已知交易回合/持仓/信号数据，断言追高占比、听模型对比、模型归因、同形态分布等数字正确。不碰网络。
- `ai_client` 在测试中 mock，不真调模型。
- 降级路径：LLM 返回 `None` → 结果仍含完整事实清单。

## 8. 新增配置项

- `ai_advisor_enabled`：总开关（默认关，配好再开）。
- `ai_advisor_provider`：`deepseek` | `claude`（默认 deepseek）。
- `ai_advisor_daily_cap`：研判卡每人每日上限（默认 200）。

## 9. 实现顺序（两功能分开做）

共享底座 `ai_client` 先行；再做交易教练（数据链最成熟：trade_rounds/model_winrate 现成）；再做研判卡（数据源多、采集面广）。每个功能各自：facts 构造器 + 单测 → 组装层 → 路由/任务 → 前端 → 校验闸 → 部署验证。
