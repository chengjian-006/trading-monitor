# 股票池预警标记设计

## 概述

在股票池网格页面上，对当天已触发预警的股票添加明显标记，包括红色数字角标、橙色行高亮、名称变色，以及可折叠的信号摘要栏。同时统一整张表格的状态颜色体系。

## 颜色体系

| 状态 | 字体颜色 | 字体粗细 | 背景色 | 边框 | 其他 |
|------|----------|----------|--------|------|------|
| 普通观察 | `#333` (inherit) | normal | 无 | 无 | — |
| 重点关注 | `#e63946` (红) | 700 | 无 | 无 | 名称前加 `*` 前缀 |
| 持仓(买入) | `#1a56a8` (深蓝) | 700 | `rgba(32, 128, 240, 0.08)` | 无 | — |
| 触发信号 | 名称 `#d35400` (橙) | 700 | `rgba(255, 120, 0, 0.12)` | 左 4px `#ff6b00` | 代码右侧红色圆形角标 |

### 组合状态优先级

- **背景色**：持仓蓝底 > 信号橙底（持仓的行即使有信号也保持蓝底）
- **左边框 + 角标**：只要有信号就显示，不受其他状态影响
- **字体颜色**：重点关注红色 > 持仓蓝色 > 信号橙色（关注是用户主动标记，优先级最高）

### 组合效果

| 组合 | 背景 | 字体 | 左边框 | 角标 |
|------|------|------|--------|------|
| 关注 + 信号 | 橙底 | 红色 | 有 | 有 |
| 持仓 + 信号 | 蓝底 | 蓝色 | 有 | 有 |
| 关注 + 持仓 | 蓝底 | 红色 | 无 | 无 |
| 关注 + 持仓 + 信号 | 蓝底 | 红色 | 有 | 有 |

## 信号角标

- 形状：16px 红色实心圆 `#ff3b00`
- 位置：紧贴股票代码右侧，稍微上移
- 内容：白色数字，显示当天该股触发的信号数量
- 字体：10px, bold

## 可折叠信号摘要栏

位置：表格底部，紧接在 NDataTable 下方。

### 结构

```
┌─────────────────────────────────────────────────┐
│ 🔔 今日预警 (N只股票，M个信号)        [展开/收起] │
├─────────────────────────────────────────────────┤
│ 002456 欧菲光    [短线买一]           10:32     │
│ 300750 宁德时代  [中线买一] [放量突破]  10:45/11:02│
└─────────────────────────────────────────────────┘
```

### 样式

- 背景：`#fffaf5`（极浅橙）
- 顶部 1px `#e8e8e8` 分隔线
- 默认展开，用户可点击收起
- 收起状态只显示标题行（"今日预警 (N只股票，M个信号)"）
- 每行显示：股票代码+名称（橙色加粗）、信号标签（橙色圆角小标签）、触发时间（灰色）
- 无信号时整个摘要栏不显示

### 交互

- 点击标题行或 [展开/收起] 按钮切换
- 折叠状态持久化到 localStorage

## 数据流

### 前端

现有 `signalStore.signals` 已包含当天所有信号，数据结构：
```typescript
interface Signal {
  code: string
  name: string
  signal_name: string
  direction: 'buy' | 'add' | 'sell' | 'reduce'
  triggered_at: string
}
```

新增 computed：
```typescript
// code -> Signal[] 映射
const signalsByCode = computed(() => {
  const map = new Map<string, Signal[]>()
  for (const s of signalStore.signals) {
    if (!map.has(s.code)) map.set(s.code, [])
    map.get(s.code)!.push(s)
  }
  return map
})
```

### 后端

无需修改。现有 `/api/signals/today` 和 WebSocket 推送已满足需求。
