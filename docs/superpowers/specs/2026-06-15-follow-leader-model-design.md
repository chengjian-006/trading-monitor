# 「看A做B」龙头跟风模型设计 — BUY_FOLLOW_LEADER

> 日期: 2026-06-15 · 分支: trade-rounds-phase1 · 状态: 设计待评审
>
> 一句话: 板块龙头涨停封板,自选池里同题材还在滞涨的票放量走强,赌跟风补涨。

## 1. 背景与动机

现有所有买卖点都是**单票看自己**: `signal_engine.detect_signals(df)` 一次只吃一只票的量价/均线就出信号,
不跨标的。而「看A做B」是另一个维度的模型——**先看全市场涨停龙头 A,再回到自选池找同板块的跟风票 B**。
维度不同,不能塞进 `detect_signals`,需要单独的跨标的扫描器。

## 2. 核心决策(已与用户确认)

| 维度 | 决策 |
|---|---|
| A→B 关系 | 龙头带跟风(同板块) |
| A 发车触发 | 龙头**涨停封板** |
| B 候选范围 | **仅自选池** |
| 选 B 口径 | **滞涨 + 放量走强** |
| 同板块判定 | **概念优先 + 行业兜底**(概念取不到时回退一级行业) |
| A 的范围 | **全市场任意封板龙头**都算 A(只 B 限自选池) |
| 上线方式 | **先做粗验证回测(Phase 0 闸门),回测站得住才进 live** |
| B 滞涨阈值 | 当前涨幅 ≤5% 且 < 龙头涨幅(已确认,不再放宽/收紧) |

## 3. 架构: 新建独立扫描器(不进 detect_signals)

方案 A(已选)。盘中周期任务 `backend/services/follow_leader_scanner.py`,每 60s:

1. 拉涨停池(复用 `fetcher/limit_pool.py`)→ 找当日**封板龙头** A 及其所属板块;
   同板块多个涨停时取**连板高度最高 / 最早封板**者为展示龙头。
2. 遍历自选池实时行情(复用 `quote_refresher` 已刷的现价/涨幅/量),筛出满足"可买跟风"的 B。
3. 命中即出 `BUY_FOLLOW_LEADER` 信号,走现有信号/推送管线(notifier + lark_card),同 B 当日只推一次。

否决的替代方案:
- B(塞进 quote_refresher 每 3s 那趟): 跨股票板块上下文太重、3s 太频会噪,仍得限到 60s,等于把 A 耦合进去更乱。
- C(盘前出一次跟风候选清单): 不符合"封板后盘中跟风"的即时性。

## 4. 模型规格 `BUY_FOLLOW_LEADER`(看A做B·龙头跟风)

- **定位**: 右侧 · 研发。
- **A 发车(信号源,全市场)**: 某板块内有票**现价涨停且封板**(复用 limit_pool 封板判定)。
  取该板块当日连板高度最高 / 最早封板者为龙头 A 展示。
- **B 可买(下单标的,仅自选池)**,需同时满足:
  1. **同板块**: B 的概念/题材命中 A 所属板块;概念取不到 → 回退一级行业(`industry` 字段,每票现成)。
  2. **滞涨**: B 当前涨幅 ≤5% 且 < 龙头涨幅。
  3. **放量走强**: 按 U 型系数推全天量 ≥ 近 10 日均量 ×1.5,且价在走强(涨幅 >0 且站上分时均价,或涨幅 ≥2%)。
- **时机**: 盘中 10:00 起,每 60s 扫;同一 B 当日只推一次(alert_throttle 节流)。
- **出场**: 沿用右侧快出族 —— +7% 卖半 / 剩半破 MA10×0.98 / −6% 止损 / T+10 时停。

## 5. 数据依赖与风险

1. **概念映射靠东财 push2**(`get_stock_concepts`),而东财 prod IP 已被封 → 盘中概念会取数失败,失败即回退行业;
   概念结果做**长 TTL 缓存**(当日复用,best-effort 刷新)。这是"概念优先+行业兜底"的实际落地形态。
2. **龙头/封板来源** = `fetcher/limit_pool.py` 涨停池,需在实现阶段核对它能给到 code + 板块 + 封板状态。
3. **行业字段** = 自选池每票现成 `industry`(一级行业,31 个),作为兜底口径零额外依赖。

## 6. Phase 0 — 粗验证回测(上线闸门)

目的: 在投入 live 工程前,先看"龙头封板 + 同板块滞涨放量"这条滤网到底有没有 edge。

- **脚本**: `backend/scripts/bt_follow.py`(对齐既有 bt_*.py 风格)。
- **数据**: `kline_cache` 全市场 5 年日线库(已有 ~4964 只);本地用 `py -3` 连库跑。
- **涨停/封板近似**: 用日线收盘价 ≈ 涨停价判当日涨停(按板: 主板 10% / 创业科创 20% / ST 5%);
  封板时点无分钟数据,用"当日涨停收盘"近似。
- **板块归属**: 历史概念取不到,**回测一期用行业判同板块**(行业归属稳定,适合粗验证);
  概念口径留到 live。
- **逻辑**: 对每个交易日 D,板块内有票涨停(龙头 A)→ 同板块其它票 B 当日滞涨(涨幅 ≤5% 且 < 龙头)
  + 放量(量 ≥ 近 10 日均 ×1.5)+ 走强(涨幅 >0)→ 次日开盘买入 → 右侧出场族跑前瞻 →
  统计胜率 / 盈利因子 / 单笔均收益。
- **对照基线**: 与"随便买同板块任一票"对比,隔离跟风滤网的增量 edge。
- **回测局限(结论须标注)**: 日线近似封板时点**偏乐观**——日内龙头封板时 B 可能已涨上去,
  实盘可得性低于回测,结论要打折看。
- **判据**: 跟风滤网相对基线有正向 edge(胜率/PF/单笔收益其一显著占优)且样本足够 → 进 Phase 1;否则不上线,记台账。

## 7. Phase 1 — Live 实现要点(回测过关才做)

- `follow_leader_scanner.py`: 注册为交易时段周期任务(60s),`_is_trading_time` 闸门,10:00 起。
- 概念优先 + 行业兜底的同板块判定,概念长 TTL 缓存。
- 命中 B → `BUY_FOLLOW_LEADER` 信号入库 + 推送(lark 卡片含龙头 A、板块、B 状态),alert_throttle 当日去重。
- 接回合记录(trade rounds)与纸面账户,积累真实样本。
