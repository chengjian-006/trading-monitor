# 观潮 · 全系统 UI 设计基线（design-baseline）

> 版本：v1.0（源自 2026-07-17 定稿的「全系统 UI 基线对标方案 v0.2」）
> 对标来源：GitHub Primer（token 体系与交互规范）+ TradingView / 富途 / 雪球（行情产品惯例）+ Vue3 组件生态调研。

## 0. 地位与适用范围

本文档是**全系统页面 UI/UE 的唯一基线**。已拍板的方向决策：

- **方向 A 先行**：全站换肤（token / 密度 / 反馈规范化），不动交互骨架，22 个页面渐进迁移，随时可停。
- **方向 B 排队**：盘中高频页升级 TradingView 式 master-detail 工作台（行点击联动右侧详情区），作为**可选批次 4**，有交互重学成本，A 完成后再议。
- **涨跌语义一次定死**：up=红=涨，down=绿=跌（A股惯例），与「危险/成功」语义**分离，互不借用**。
- 配套代码落地在 `frontend/src/styles/tokens.css`。
- 与 `docs/push-design-baseline.md`（飞书推送卡基线）互为姊妹篇：**页面归本文档，推送卡归彼文档**；两侧共用的图形词汇（胜率条/战绩热图/KPI 大数字）保持 1:1 同款。

适用范围：全部现有页面（22 个）+ 未来新增页面。**以后新开页面先认类型再动手**（见第 1 节），不再一页一个样。

## 1. 页面场景类型全景（9 型收编全系统）

全系统 22 个现有页面 + 可预见的未来页面，全部归入 9 种场景类型（TYPE A~I），另加 TYPE J 移动端形态横切所有类型。每型固定一套「版式骨架 + 密度档位 + 组件配方 + 交互规则」。所有页面统一浅色基线风格，深色主题由同一套 token 自动切换。

### TYPE A · 盯盘工作台（实时高密度）

**页面归属**：监控看板 / 股票池 / 问财候选榜；未来：题材低吸榜。

**版式骨架**：市场概览条（固定首行）+ 紧凑表格（28~36px 行）。

**规则**：

1. **密度用紧凑档 28~36px、12px 字号**——盯盘生产力 = 一屏行数。
2. **市场概览条固定首行**（指数 + 涨跌家数，富途首屏惯例）。
3. **实时三件套**：tabular-nums 零抖动 / 单元格 flash / 盘中 3s 轮询切后台自动暂停。
4. 行点击联动详情（方向 B 阶段升级右侧详情区），操作确认才弹窗。
5. 虚拟滚动必开（>50 行）；筛选胶囊态存 localStorage。

### TYPE B · 查询列表页（筛选 + 翻页）

**页面归属**：信号历史 / 预警总览 / 操作日志；未来：消息/通知中心。

**版式骨架**：筛选条 + 标准表格（36px 行）+ 计数页脚。

**规则**：

1. **筛选条永远在表格上方一行**：时间范围 + 枚举下拉 + 搜索框，选中态高亮，筛选态存 localStorage（回来还在）。
2. **标准密度 36px**；长列表虚拟滚动或分页，页脚必带总数。
3. **导出按钮固定右上**（全站统一位置）。
4. 行点击 → 右侧抽屉看详情（不跳页不弹窗）。
5. 空结果给空状态五件套：为什么空 + 一键清筛选。

### TYPE C · 分析报告页（卡片流）

**页面归属**：买卖胜率 / 涨停复盘 / 人气分析 / 交易分析 / 问财观点；未来：自动周报/月报。

**版式骨架**：KPI 三栏 → 图表卡 → 结论卡 → 明细表（48px 宽松）。

**规则**：

1. **信息顺序定死：结论先行**——KPI 大数字 → 图表 → 结论卡 → 明细下沉（雪球式分层披露）。
2. **一卡一主题**，卡片流单列纵向滚动；宽松密度 48px。
3. 统计图表统一 echarts 规范（按需注册），K线/分时仍走 lightweight-charts。
4. 专业口径进 tooltip（主界面留大白话，术语 hover 出定义）。
5. 问财观点等内容流：卡片 + 折叠长文，同属此型。

### TYPE D · 账户资产页（资产头 + 持仓）

**页面归属**：模拟账户；未来：实盘账户视图。

**版式骨架**：资产大数字头 → 权益曲线 → 持仓表。

**规则**：

1. **资产头 = KPI 大数字（26px/700/tabular-nums）**，今日盈亏红绿直显。
2. 权益曲线用 lightweight-charts，可与基准（沪深300）叠加对比。
3. 持仓表标准 36px，盈亏列 tabular 右对齐 + 语义色加粗。
4. 流水/成交记录切页签（改 URL），不堆同屏。

### TYPE E · 工具工作页（参数 → 运行 → 结果）

**页面归属**：模型回测；未来：自定义选股器 / 参数调优台。

**版式骨架**：参数区吸顶 → 运行反馈 → 结果 KPI + 明细。

**规则**：

1. **参数区一行吸顶**（NAffix），改参数不立即跑，显式点「运行」。
2. **>3s 必上进度条**（定型指示器），**>10s 转后台任务 + 完成通知**——绝不让用户盯 spinner 干等。
3. 结果结构与分析报告页（TYPE C）同配方：KPI → 图 → 明细。
4. 参数组合存 localStorage，回来即恢复上次实验。

### TYPE F · 图鉴 / 目录页（卡片网格）

**页面归属**：模型图鉴 / 推送模版预览 / 版本更新；未来：帮助/术语表。

**版式骨架**：网格卡（果仁式策略卡：KPI + 胜率条 + 战绩热图）。

**规则**：

1. **网格卡片**（2~3 列自适应），卡内结构统一：名称 + 节奏标签 → 胜率条 → 战绩热图。
2. 点卡展开完整详情（触发链/参数/回测口径），当页展开不跳转。
3. 与推送卡战绩同口径互跳（推送基线文档既有约定延续）。
4. 版本更新页 = 时间线变体：同卡片语言，左侧时间轴。

### TYPE G · 配置 / 表单页（分组卡 + 即改即存）

**页面归属**：策略配置 / 系统设置 / 定时任务 / 用户管理 / 推送偏好；未来：推送偏好中心。

**版式骨架**：分组卡片 + 滑动开关 + 危险区隔离。

**规则**：

1. **单项设置即改即存**（开关点一下、输入失焦自动保存 + 「已保存」轻提示）；**多字段成组表单才用显式保存钮**——两种模式不混用。
2. 开关行三件套：名称 + 灰色小字说明 + 右侧 switch。
3. **危险操作红色隔离区**：红框卡 + 确认弹窗（320px、红色确认键、具体文案如「清空 24 条信号」）。
4. 管理列表（用户/任务）：标准表格 + 行内状态 switch + 行尾单操作外露。

### TYPE H · 详情层（抽屉 / 整页详情）

**页面归属**：个股详情（全局弹层）/ 分时整页 / 信号详情 / 回合归因。

**版式骨架**：右侧抽屉 = 标的头 + 图表 + 字段组 + 操作行。

**规则**：

1. **浏览详情一律右侧抽屉**（列表上下文保持可见）；抽屉宽度两档 480/640px。
2. 抽屉内结构统一：标的头 → 图表 → NDescriptions 字段组 → 底部操作行（≤3 个）。
3. 需要沉浸的（整页分时）才用整页路由，带面包屑返回。
4. **弹窗（Modal）只留给操作确认**，320px 档，永不用于浏览内容。

### TYPE I · 登录 / 引导页

**页面归属**：登录；未来：首次使用引导；全局空状态（横切）。

**版式骨架**：浅色居中卡（深色渐变退役）+ 空状态五件套。

**规则**：

1. 登录页换**浅色统一**（当前深色渐变退役），品牌蓝点缀即可。
2. **空状态五件套全站统一**：图标 → 为什么空 → 一句描述 → 主操作 → 帮助链接；分「首次使用 / 暂时没数据 / 出错了」三种文案口径。
3. 错误态说人话：「行情源超时，已自动切换备源重试」优于「加载失败」。

### TYPE J · 移动端形态（横切所有类型）

**页面归属**：全部页面的 <768px 形态（768 断点契约，既有硬规范延续）。

**版式骨架**：宽表 → 卡片化 · 底部 Tab。

**规则**：

1. **768 断点契约不变**：useResponsive + mobile.css 单一真相源。
2. **宽表一律卡片化**（ResponsiveTable）：首行 = 名称 + 现价涨幅，次行 = 辅助信息灰字 + 标签。
3. 操作沉底部：悬浮按钮（NFloatButton）/ 底部抽屉替代 hover 菜单。
4. 长页标配 NBackTop；筛选器收进底部抽屉。
5. **每个新页面上线前移动端过一遍是硬闸**（既有规范延续）。

## 2. 设计 Token 基线（全部类型共用）

三段式功能命名（用在哪 × 表达什么 × 强度）。A股语义映射一次定死：**up=红=涨，down=绿=跌**，与「危险/成功」语义分离、互不借用。全站 395 处硬编码颜色全部收敛到本表。落地文件：`frontend/src/styles/tokens.css`。

### 2.1 色板（浅色主题值；深色主题另有一套，由同名 token 切换）

| Token | 值 | 用途 |
|---|---|---|
| `--accent` | `#0969DA` | 强调 / 链接 |
| `--up-fg` | `#D92B26` | 涨 · 文字/线 |
| `--up-bg-muted` | `rgba(217,43,38,.09)`（9% 透明） | 涨 · 衬底 |
| `--down-fg` | `#0F8A5F` | 跌 · 文字/线 |
| `--down-bg-muted` | `rgba(15,138,95,.09)`（9% 透明） | 跌 · 衬底 |
| `--warn-fg` | `#9A6700` | 警示 |
| `--fg-default` | `#1F2328` | 文字 · 主 |
| `--fg-muted` | `#656D76` | 文字 · 次 |
| `--fg-subtle` | `#8B949E` | 文字 · 辅 |
| `--bg-default` | `#F6F8FA` | 页面底 |
| `--bg-surface` | `#FFFFFF` | 卡片面 |
| `--border-default` | `#D1D9E0` | 边框 |

**emphasis / muted 两档规则**：每个语义色固定两档——

- **实底 emphasis**（配白字）：只给主按钮和最高级预警；
- **浅底 muted**（配深色字）：信号标签 / 徽章 / 热度块全走这档。

这样页面才不会一片实色块。涨跌色刻意压饱和（禁纯红纯绿，红绿明度接近），红/绿/灰平三态齐全；红绿是屏幕上唯一语义色，其余装饰全灰化。

### 2.2 字号阶梯（去掉 zoom 1.25 全局缩放 hack，按真实尺寸定准）

| Token | 规格 | 用途 |
|---|---|---|
| `--text-page-title` | 20px / 600 | 页面标题 |
| `--text-section` | 16px / 600 | 区块标题 |
| `--text-body` | 14px / 400 | 正文与表单 |
| `--text-table` | 12px / 400 | 表格与辅助信息 |
| `--text-kpi` | 26px / 700 / tabular-nums | KPI 大数字 |

### 2.3 间距、圆角、字重、阴影

- **间距只允许 4 / 8 / 12 / 16 / 24px**（4px 网格）。
- **圆角两档**：6px（卡片/按钮）+ 3px（表内小件）。
- **字重最高 600**，不用 700 以上（KPI 大数字 700 是唯一例外）。
- **阴影两档**：静置卡片微影 / 浮层描边 + 双层软影。

### 2.4 表格密度三档（可切换）

| 档位 | 行高 | 适用 |
|---|---|---|
| 紧凑 | 28px | 榜单 / 盯盘（盘中页默认） |
| 标准 | 36px | 默认 |
| 宽松 | 48px | 复盘报告 |

## 3. 交互与操作标准（一次定死，全站执行）

### 3.1 加载分流（按预计时长）

- **<1s**：什么都不显示（闪 spinner 反而干扰）。
- **1–3s**：内容区 skeleton 保形状，小按钮 spinner；相邻加载区合并成一个指示器。
- **3–10s**：进度条（定型指示器）。
- **>10s**：转后台任务 + 完成后页顶横幅通报。

### 3.2 反馈去 toast 化

- **成功靠界面自证**：静音后按钮态立即变、导入后行直接出现，不再弹「操作成功」。
- **失败必须就地、持久提示**：贴着出错位置，不自动消失。
- **破坏性操作统一确认弹窗**：红色确认键 + 具体文案（「删除 40 条交割记录」而非「确定」）。

### 3.3 表格五规

1. 数字列右对齐 + tabular-nums + 全列统一小数位（实时刷新零抖动的行业第一硬规则）。
2. 表头：灰底 + 次要色 + 600 字重，字号不放大。
3. 弃斑马纹；行分隔 1px；整行 hover 半透明灰。
4. 变动单元格 flash 550ms（约 10% 透明度语义色衬底淡出），只闪变动的格子不闪整行，动画中不重复触发。
5. 行操作 ≤1 个外露，其余收进「···」菜单。

### 3.4 导航与详情

- 改 URL 的用页签/侧栏；只切面板的用分段控件。
- 看详情：行内展开或右侧抽屉；**弹窗只留给操作确认**。
- 空状态五件套：图标 + 为什么空 + 一句描述 + 主操作 + 帮助链接。
- 移动端 768 断点契约不变：宽表卡片化，操作沉底部。
