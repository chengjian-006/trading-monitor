# 观潮·智能监控系统 合规基线

> 版本 v1.0 · 2026-07-17 · 适用范围：官网（site/）、后台前端（frontend/）、推送通道（飞书/微信）、对外注册流程
> 性质：内部执行基线，不是法律意见书。开放收费或规模化注册前，应请执业律师复核一次。

## 0. 定位声明（一切合规判断的出发点）

本系统是**技术工具**：按用户设定或公开披露的规则扫描行情、命中即提示，并提供历史回测与交易复盘功能。
它**不是**证券投资咨询服务——不荐股、不代客理财、不预测涨跌、不接券商交易接口、不碰用户资金。

依据《证券法》第 120 条、第 160 条及《证券投资顾问业务暂行规定》，未持牌机构不得从事"就证券投资提供分析、预测或建议"的经营性服务。工具与投顾的分界线在于：**是给出"某规则被触发"的客观事实，还是给出"应当买卖某证券"的判断**。本文档的全部红线都是为了让产品稳稳站在事实一侧。

## 1. 证券合规红线清单

既定口径红线（已写入 site/src/content.ts 头注释，全站文案的总纲）：

1. 定位是工具，不是荐股服务；
2. 不在公开页展示胜率、收益率数字——讲方法论，不讲战绩；
3. 公开证伪记录——连失败的回测也公开，用透明换信任。

以下按三档展开为可执行清单。

### 1.1 绝对禁止（任何页面、任何推送、任何客服话术都不得出现）

**表述禁区：**

| 禁止类型 | 示例（均不得出现） | 原因 |
|---|---|---|
| 保证收益 | "稳赚""必涨""保本""跑赢大盘""年化 XX%" | 《证券法》明令禁止承诺收益 |
| 荐股表述 | "推荐买入 XX 股""今日金股""建议建仓/加仓某股" | 构成无牌照投资建议 |
| 预测涨跌 | "明天大概率涨""目标价 XX 元""还有 30% 空间" | 预测类表述是投顾业务的典型特征 |
| 诱导交易 | "最后上车机会""不买就错过""跟着信号操作即可躺赢" | 诱导性营销 + 变相荐股 |
| 冒充资质 | "专业投顾团队""持牌分析师""与 XX 券商合作" | 虚假宣传，且暗示投顾资质 |
| 代客含义 | "帮你操作""托管账户""自动跟单下单" | 代客理财红线；系统本就不接交易接口 |

**数据展示禁区（公开页面 = 官网、未登录可见页、对外分享物料）：**

- 不展示任何模型的胜率、收益率、盈亏比、PF 等战绩数字（登录后的内部页面可展示，见 1.3）；
- 不展示"跟随信号可获得 XX 收益"的任何形式换算；
- 不展示具体真实股票代码 + 买卖方向的组合作为营销示例（官网示例必须用虚构代号，现行 `XXXXXX 示例股` 的做法保持）；
- 不排"收益排行榜"、不搞"晒单"类传播素材。

**行为禁区：**

- 不提供一对一的个股买卖问答（用户问"XX 股能不能买"，标准回复口径："本工具不提供个股建议，只能告诉你它触发了哪些规则条件"）；
- 不开设收费荐股群、不以任何名义收取"服务费换股票建议"；
- 不接入券商交易接口、不代为下单、不索取券商账号密码（官网 FAQ 已明示，保持）。

### 1.2 需要免责声明伴随（可以做，但出现处必须挂免责或口径说明）

- **信号推送**：每条推送卡片必须带"口径与免责"段（现行折叠式可接受，但不得删除）；
- **回测数据**（登录后页面）：展示历史胜率/收益的页面，须在同屏标注"历史回测统计，不代表未来收益"及统计口径（样本区间、成交假设、费用扣减）；
- **AI 研判/持仓晚报**：保留现行"AI研判仅供参考，最终决策在你"文案，不得弱化；
- **证伪记录与方法论页**：可公开，但必须保留尾部风险披露（现行"回测样本未覆盖单边熊市"的 caveat 是合规资产，不得删）；
- **第三方内容转载**（博主观点、问财结果等）：须标注来源并附"观点仅供参考，别直接跟单"类提示（现行做法保持）。

### 1.3 允许（无需额外声明，但须守住措辞规范）

- 陈述客观规模事实：覆盖标的数、K 线条数、扫描频率、模型个数、版本迭代次数（官网 STATS 现行口径，全部可核对）;
- 描述规则逻辑本身："收盘价低于 MA10 达 2.4%，14:30 后确认"——这是事实陈述；
- 登录后向注册用户展示模型战绩、回测明细（属工具功能而非公开营销，前提是带 1.2 的口径标注）；
- 用户自设规则、自导交割单、自行复盘的全部功能。

### 1.4 信号措辞规范（写文案、写推送模板时逐条对照）

| 禁用措辞 | 规范措辞 |
|---|---|
| 建议买入 / 可以买了 | 触发「XX」买点规则条件 |
| 建议卖出 / 赶紧跑 | 触发「XX」卖出纪律条件 / 破位警戒 |
| 看涨 / 要启动了 | 命中规则：缩量回踩 MA10 后放量突破昨高 |
| 目标价 XX | （不给目标价；如展示用户自设的交易计划价，标注"你设定的计划价"） |
| 止损位 XX，建议执行 | 触及你设定的止损纪律线 / 处置：按你设定的破位纪律处理 |
| 强烈推荐关注 | 进入「临近买点榜」/ 满足 X 项条件中的 Y 项 |

核心句式：**主语是规则，不是我们**。"规则被触发"而非"我们建议"；"按你设定的纪律"而非"你应该"。

## 3. 用户协议与隐私政策提纲（开放注册前必须具备）

对照《个人信息保护法》的最小合规集。以下为提纲级模板，成文时按此填充。

### 3.1 用户协议提纲

1. **服务性质**：技术工具定位声明（引用第 0 节口径）+ 标准免责全文；
2. **账号规则**：注册条件、账号安全责任、禁止转借/共享收费账号；
3. **用户行为边界**：不得将推送内容用于对外荐股/收费转发；不得爬取、转售系统数据；
4. **知识产权**：模型规则、回测结论、页面内容归运营方；用户导入的交割单数据归用户；
5. **服务变更与终止**：模型可能上线/下线/调参（引证伪机制），不构成违约；
6. **责任限制**：因行情数据源中断、延迟导致的信号缺失不承担投资损失责任；
7. **争议解决**：适用法律与管辖。

### 3.2 隐私政策提纲（PIPL 最小集）

1. **收集哪些信息**（据实列举，不写"包括但不限于"）：
   - 注册身份：手机号或微信号、昵称；
   - 用户主动导入：券商交割单（含成交记录，属敏感财务信息，须单独提示）；
   - 用户配置：自选股、持仓、交易计划、推送偏好；
   - 日志：登录时间、IP、操作记录（安全审计用）。
2. **每项用途**：一一对应写明（手机号→登录与推送触达；交割单→复盘归因，仅本人可见；不用于任何画像营销）；
3. **存储**：存储地点（境内云服务器）、期限（账号存续期间；注销后 X 日内删除或匿名化）；
4. **共享与转移**：不向第三方出售；推送经由飞书/微信通道传输须列明（第三方 SDK/接口清单）；
5. **用户权利**：查询、更正、导出、删除（交割单可自行删除）、注销账号的入口和时限；
6. **未成年人**：不面向未满 18 周岁用户提供服务；
7. **变更通知**：政策更新的告知方式。

### 3.3 落地要求

- 注册流程必须"勾选同意 + 可点开全文"，默认不勾选；
- 交割单导入功能首次使用时单独弹一次敏感信息提示（一次性，不重复骚扰）；
- 提供注销入口（可先做人工注销，但协议里承诺的时限必须能兑现）。

## 4. 数据来源合规

行情与公告数据来自新浪、腾讯、巨潮资讯、东方财富等公开接口。规则：

1. **来源标注**：对外展示行情/公告数据的页面和推送，标注"数据来源：新浪财经/腾讯财经/巨潮资讯网等公开渠道，可能存在延迟"；官网如引用规模数字无需逐条标注，但被问询时口径可回溯；
2. **用途限定**：抓取数据仅用于本系统的展示、信号计算与回测分析，**不得转售原始行情数据**、不得提供原始数据批量下载/导出接口（用户自己的交割单导出除外）、不得对外提供行情 API 转发服务；
3. **抓取克制**：控制请求频率，不绕过对方的访问控制措施；某源封禁（如东财对生产 IP）即切换备源，不做对抗性绕过；
4. **不担保准确性**：免责声明及用户协议中写明"行情数据来自第三方公开渠道，可能延迟、缺失或错误，不作准确性担保"；
5. **公告与新闻**：转述公告内容注明来自巨潮/交易所公开披露；不篡改、不断章取义制造情绪化标题。
